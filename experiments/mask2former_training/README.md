# Mask2Former Training (CubiCasa5k)

How the geometry model in the hybrid pipeline was trained. Resulting weights:
`model/extracted/model_final.pth` (used by `pipeline/mask2former_infer.py`).

## Dataset

- CubiCasa5k (Finnish residential), images `F1_scaled.png`
- 5 classes (0-indexed): room, wall, door, window, railing
- Split: 4200 train / 253994 annotations; val + test separate
- Format: COCO JSON in `dataset/{train,val,test}.json`, generated by `build_dataset.py`
  from CubiCasa SVGs (`model.svg` polygons converted to RLE)
- `dataset/images` is a symlink to `~/Downloads/cubicasa5k`

## Architecture

- Mask2Former, backbone Swin-B (EMBED_DIM 128, depths [2,2,18,2], num_heads [4,8,16,32], window 12, pretrain 384, drop_path 0.3)
- MaskFormerHead, MSDeformAttn pixel decoder, 100 object queries, 10 decoder layers
- Config set programmatically (no YAML). See `evaluate.py::build_predictor()` and `sagemaker/train.py` for full config keys.

## Training

- Hardware: Prime Intellect, 1x A100 80GB SXM4
- Hyperparams: batch 2, max_size 1600, AdamW, LR 1e-4, cosine + warmup, 20000 iters
- VRAM: ~34GB. Speed: ~1.4s/iter, total ~7.8h
- Environment: torch 2.4.0+cu124, detectron2 from source, gcc-12, CUDA 12.4, MSDeformAttn compiled

## CPU inference (local Mac)

`vendor/Mask2Former/.../ops/functions/ms_deform_attn_func.py` has the `HAS_CUDA_MSDA` flag
and falls back to the pure-Python implementation when CUDA is absent.
This patch is required; do not revert it.

## Files

- `build_dataset.py` - CubiCasa SVG to COCO JSON (5 classes)
- `evaluate.py` - COCO mAP on test set (run: `python3 experiments/mask2former_training/evaluate.py`)
- `sagemaker/` - AWS SageMaker training variant (abandoned due to infra failures, kept for reference)
- `dataset/` - COCO JSONs + images symlink
